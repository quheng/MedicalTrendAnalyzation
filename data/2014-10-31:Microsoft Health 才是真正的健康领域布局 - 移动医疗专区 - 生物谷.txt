Microsoft Health 才是真正的健康领域布局 - 移动医疗专区 - 生物谷
关于Microsoft Band，我们联系到与这个项目相关的工作人员。从这位“佚名”的回答中，我们可以一窥有关 Microsoft Band 定位和目标方面的信息。Google在一天之前发布了在 UX 方面可以接近Apple Watch官方应用Workout的Google Fit个人健康数据服务。微软在仅一天之后就低调发布 Microsoft Band 和 Microsoft Health 服务不免引人联想。但这位工作人员告诉我们——这样的“撞车”只是巧合。微软的Microsoft Band和 Microsoft Health服务一直在筹备当中，只是为了保证一切准备就绪，以便让用户可以随时购买体验。对于穿戴式设备，Apple和 Google都已经坚定地直接迈入“全功能智能腕上设备”领域，Apple Watch和 Android Wear都可以独立安装应用，并脱离手机实现一部分功能。另一方面，类似 Jawbone UP、FitBit各系列，及 Misfit 等无法独立使用的手环类产品一直没有得到市场的广泛认可，微软此时推出类似的手环似乎是在一个错误的时间站了错误的队。而在这一方面，他并不愿意告诉我微软的智能手表计划（虽然前诺基亚现微软芬兰Tampere设备部门、前诺基亚现微软北京实验室和微软雷德蒙德硬件部门都有相关项目）。但他承认新产品门类确实需要经过市场的检验，并且之前很多类似产品的市场反馈都不甚理想，即使是微软进入这个领域也需要很谨慎地对待。微软进入Nadella时代之后，硬件产品的各方面策略就开始明显向企业市场转型。面对XBOX和Lumia系列Windows Phone当下的疲软，Surface Pro 3 在微软新财季报中的强势表现就是 Nadella 这一策略方向正确的佐证。但 Microsoft Band 显然是又一款完全面向个人用户的日常消费产品，与近期微软对外强调的“强化企业各环节生产力”主题并不相符。有趣之处在于，由微软健康方面项目负责人之一、原 Xbox 部门创始人 Todd Holmdahl 在官方博客发表的文章中，Microsoft Band的笔墨有限，甚至像是 Microsoft Health的附属产品。这样一来，联系微软近期在消费级市场的多平台并行策略，就能够理解Microsoft真正的意图所在。微软长久以来一直提供有名为 HealthVault的个人健康信息接入和管理工具，这项工具可以与 MSN 健康应用同步信息，并能导入个人社保卡数据（限美国）。Microsoft Health 的推出，表面上看来一下将微软在个人健康方面的服务增加到三个，但它更像是对HealthVault的全面取代。HealthVault 存在多时但并未被用户熟知， 此时将它简单更新并推广到其它平台是不具备说服力的。Microsoft Health随 Microsoft Band 手环推出，话题方面抓足眼球，同时迅速在 Android、iOS、Windows Phone甚至 Mac OS X 应用商店上架（比 Microsoft Band 正式上市时间早一天），支持多款第三方健康软硬件且能和 HealthVault同步信息。这样的新品牌才有机会真正占领用户市场。连 TheVerge 微软观察家 Tom Warren也在 Twitter上毫不避讳地说：“Microsoft Band 最好的一点在于无论是宣传还是手环本身都完全不带 Windows 标识”，一个脱离旧形象的好产品能够帮助微软重塑 Microsoft 这个最重要的主品牌。而SDK的推出，是在官方合作之外等待更多可能性，利用开发者和终端用户的相互促进，把Microsoft Health打造成一个可自持续演进的大健康数据平台。在接受采访的人看来，从智能手机开启的消费数码电子时代开始，单一功能的产品就已经无法满足需求。同理，单一品类也无法构成足够说服消费者的服务体系。无论是软件还是硬件，产品都应该更注重整个生态系统的联动体验。再次之上考虑鼓励开发者进行更多创造，让用户的生活真正达到更轻松，更一体化而不至于体验分裂。到这里，今天微软新品的定位脉络已经很清晰：Microsoft Health是真正唱主角的台柱子，Microsoft Band 是借着这个健康领域的正式进军对新品类市场进行试探。在消费市场上，微软依然坚持将自己的产品平台化：接入更多的服务、设备和数据，用自己的云端优势在此之上为不同平台用户带来价值。(生物谷Bioon.com)
